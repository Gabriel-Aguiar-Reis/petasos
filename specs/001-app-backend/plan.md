# Implementation Plan: App Backend — Clean Architecture Domain & Data Layer

**Branch**: `feat/001-app-backend` | **Date**: 2026-04-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-app-backend/spec.md`

## Summary

Build the domain and data layers for RoadLedger (MVP): typed domain entities, repository interfaces, 13 use cases, Drizzle ORM schemas, and SQLite-backed repository implementations — all fully offline, framework-free at the domain boundary, and exercisable with in-memory stubs. Covers trip/earnings recording, cost logging, fuel tracking (per fuel type), dashboard summary, JSON export, work sessions, goals, and quick-entry mode (nullable optional fields on CreateTrip/CreateCost).

## Technical Context

**Language/Version**: TypeScript 6.x — strict mode, no `any`  
**Primary Dependencies**: React Native (Expo SDK 55.0.15), `expo-sqlite` 55.0.15, `drizzle-orm` 0.45.2, `drizzle-kit` 0.31.10, `uuid` 13.0.0, `zod` 4.3.6, `zustand` 5.0.12, `@tanstack/react-query` 5.99.0, Jest 29.7.0, `ts-jest` 29.4.9  
**Storage**: SQLite via `expo-sqlite` + Drizzle ORM (local only, offline-first)  
**Testing**: Jest + `ts-jest` — domain use cases tested with in-memory repository stubs (Node environment, zero device dependency)  
**Target Platform**: Android only (MVP) — Expo managed workflow  
**Project Type**: Mobile app (React Native / Expo)  
**Performance Goals**: Full workday dataset (10 trips, 5 costs, 1 fuel log, 1 session) stored and retrieved in < 200 ms on average Android device (SC-003)  
**Constraints**: 100% offline; no network calls in domain or data layers; domain layer has zero imports from Expo/React Native/SQLite; monetary values rounded to 2 decimal places  
**Scale/Scope**: Single-user, single-device MVP; ~300 trips + 150 costs + 30 fuel logs per 30-day export window
