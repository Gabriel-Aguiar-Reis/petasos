# Implementation Plan: Backend Domain Refactor — Extended Domain Model & Pure Functions

**Branch**: `004-backend-domain-refactor` | **Date**: 2026-04-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-backend-domain-refactor/spec.md`

## Summary

Extend the domain and data layers built in `001-app-backend` with 14 new/updated entity types, 5 pure domain functions, ~40 use cases, 15 new Drizzle tables, and 3 application-layer services. Key additions: lookup entities (FuelType, VehicleType, TripPlatform), expanded Vehicle, `Trip.platform` → `Trip.platformId` FK migration, recurring Costs (RFC 5545 via rrule.js in app layer), FuelConsumptionRecord + FuelPriceRecord + MileageRecord, Profit, updated WorkSession, Reminder, Maintenance, SpecialDay (BrasilAPI sync), PlannedAbsence, TripOfferRecord + Trip Offer Pill evaluation, SavedProfitGoal, PlatformProfitGoal, and UserSettings. The domain layer remains dependency-free; rrule.js lives exclusively in `src/application/services/`.

## Technical Context

**Language/Version**: TypeScript 6.0.2 — strict mode, no `any`
**Primary Dependencies (existing)**: React Native 0.83.4 (Expo SDK 55.0.15), `expo-sqlite` 55.0.15, `drizzle-orm` 0.45.2, `drizzle-kit` 0.31.10, `uuid` 13.0.0, `zod` 4.3.6, `zustand` 5.0.12, `@tanstack/react-query` 5.99.0, Jest 29.7.0, `ts-jest` 29.4.9, `expo-notifications` 55.0.19
**New Dependency**: `rrule` ^2.8.1 — application/infrastructure layer only; zero domain imports
**Storage**: SQLite via `expo-sqlite` + Drizzle ORM (local only, offline-first)
**Testing**: Jest + `ts-jest` — domain use cases tested with in-memory repository stubs
**Target Platform**: Android only (MVP) — Expo managed workflow
**Project Type**: Mobile app (React Native / Expo)
**Performance Goals**: Consistent with 001 — full workday dataset stored and retrieved in < 200 ms
**Constraints**: 100% offline-first; BrasilAPI sync is additive/non-blocking; domain layer has zero imports from Expo/React Native/SQLite/rrule; monetary values rounded to 2 decimal places
**Scale/Scope**: Single-user, single-device MVP; extended entity set supporting all driver operational data

| II. Driver-Centric UX | ✅ PASS | `GetLastMileageRecord` pre-fills forms. `EvaluateTripOffer` delivers instant profit signal. `GetUserSettings` defaults ensure zero-config startup. |

| VII. Simplicity & YAGNI | ✅ PASS | No sync abstractions, no GPS, no cloud. `FuelLog` kept for backward compat (not removed). Repository pattern already justified in 001. |
